📖 Understanding CNNs: The "What, Why, How, Where"

❓ WHAT are CNNs?

Convolutional Neural Networks are deep learning architectures that automatically learn spatial hierarchies of features. Composed of convolutional layers, pooling layers, and fully-connected layers.

🧩 conv → ReLU → pool → conv → ReLU → pool → FC
🎯 WHY CNNs for Vision?

Traditional ML needs manual features (HOG, SIFT). CNNs learn edges → textures → parts → objects automatically. Parameter sharing & translation invariance make them extremely efficient for images.

⚙️ HOW do they work?

• Convolution: filters slide over image → feature maps.
• Pooling: downsamples (max/average) → reduces dimensions.
• Activation (ReLU): introduces non-linearity.
• Backpropagation learns filter weights.

🌍 WHERE used?

Object detection (YOLO), segmentation (UNet), medical imaging, facial recognition, autonomous driving, image search, and any visual recognition task.

🧪 Key intuition: Early layers detect edges/blobs, middle layers detect textures/patterns, deeper layers detect object parts (eyes, wheels). Pre-trained models already have these rich features!

🧬 Core Building Blocks: Conv + Pooling + Activation

🟦 Convolution

Filter (kernel) slides across input, computes dot product. Output = feature map. Learns spatial patterns.

# conceptual: 3x3 kernel on 32x32 image
output = conv2d(input, kernel, stride=1, padding=0)
📉 Pooling (MaxPool)

Reduces spatial size, keeps dominant features. Common: 2x2 window with stride 2 → halves dimensions.

# PyTorch: nn.MaxPool2d(kernel_size=2, stride=2)
⚡ Activation (ReLU)

f(x) = max(0,x). Adds non-linearity, helps gradient flow. After every conv layer.

x = torch.relu(conv_output)
💡 Why these three together? Conv extracts features → ReLU introduces non-linearity → Pooling reduces computation & overfitting. Stack them to create deep networks.

🏋️ Loading Pre-trained Models (ResNet18, MobileNet)

Pre-trained models trained on ImageNet (1.2M images, 1000 classes). They act as powerful feature extractors. Transfer learning = reuse these weights for custom tasks.

🟢 ResNet18 (Residual Network)

18 layers deep with skip connections (residual blocks). Solves vanishing gradient, high accuracy.

import torchvision.models as models
resnet18 = models.resnet18(pretrained=True)
resnet18.eval() # inference mode
📱 MobileNetV2 (Lightweight)

Depthwise separable convolutions → efficient for mobile/edge. Good speed vs accuracy tradeoff.

mobilenet = models.mobilenet_v2(pretrained=True)
mobilenet.eval()

# Complete MobileNet classifier for custom images
import torch
import torchvision.models as models
import torchvision.transforms as transforms
from PIL import Image
import requests
import json

# 1. load model
model = models.mobilenet_v2(pretrained=True)
model.eval()

# 2. ImageNet labels (download mapping)
labels_url = "https://raw.githubusercontent.com/pytorch/hub/master/imagenet_classes.txt"
classes = requests.get(labels_url).text.splitlines()

# 3. preprocessing
preprocess = transforms.Compose([
transforms.Resize(256),
transforms.CenterCrop(224),
transforms.ToTensor(),
transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
])

def classify_image(image_path):
img = Image.open(image_path).convert('RGB')
input_tensor = preprocess(img).unsqueeze(0)
with torch.no_grad():
output = model(input_tensor)
probs = torch.nn.functional.softmax(output[0], dim=0)
top3 = torch.topk(probs, 3)
results = [(classes[idx], probs[idx].item()) for idx in top3.indices]
return results

# classify your 10 custom images
for img_path in ["dog.jpg", "coffee_mug.jpg", "bicycle.jpg", ...]:
predictions = classify_image(img_path)
print(f"{img_path}: {predictions[0][0]} ({predictions[0][1]*100:.1f}%)")
🖼️ Example output: "test_dog.jpg → Siberian husky (87.4%) | malamute (5.2%) | wolf (2.1%)"

🛠️ Tools: PyTorch vs TensorFlow/Keras

🔥 PyTorch (dynamic, research-friendly)

Define-by-run, Pythonic, great debugging. Preferred for research & custom architectures.

model = models.resnet18(pretrained=True)
out = model(x) # immediate execution
🧠 TensorFlow/Keras (production, high-level)

Sequential API, easy deployment, TF Serving. Keras makes prototyping fast.

from tensorflow.keras.applications import MobileNetV2
model = MobileNetV2(weights='imagenet')

Recommendation for this course: PyTorch — because of flexibility and wide adoption in computer vision research. But both work perfectly.

📌 Quick Reference & Best Practices

  • 🔹 Preprocessing is critical – Always normalize with ImageNet mean/std when using pre-trained weights.
  • 🔹 model.eval() disables dropout/batch norm stats – mandatory for inference.
  • 🔹 Use torch.no_grad() to disable gradient computation → faster inference & less memory.
  • 🔹 For custom datasets: fine-tune last few layers (transfer learning) for better performance.
  • 🔹 MobileNet → best for real-time / edge; ResNet → higher accuracy but slower.
