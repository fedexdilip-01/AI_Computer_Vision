📖 The "What, Why, How, Where" of Feature Detection

❓ WHAT

Features = distinctive patterns like corners, blobs, or keypoints that remain detectable under rotation, scaling, or illumination changes. Harris, Shi-Tomasi corners; SIFT and ORB descriptors.

🎯 WHY

Match same object/scene across images, build panoramas, track objects, 3D reconstruction. Features act as visual fingerprints — robust to viewpoint changes.

⚙️ HOW

Detect interest points → compute descriptors (numerical vectors) → compare descriptors using BFMatcher or FLANN → geometric verification (Homography).

🌍 WHERE

Panorama stitching, AR markers, image registration, visual SLAM, object recognition, medical image alignment, satellite image merging.

📍 Corner Detection: Harris & Shi-Tomasi

📐 Harris Corner

cv2.cornerHarris(gray, blockSize, ksize, k) → detects corners by analyzing gradient intensity variations.

gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
dst = cv2.cornerHarris(gray, 2, 3, 0.04)
img[dst>0.01*dst.max()] = [0,0,255] # mark corners in red
🎯 Shi-Tomasi (Good Features to Track)

Improves Harris: returns N strongest corners. cv2.goodFeaturesToTrack()

corners = cv2.goodFeaturesToTrack(gray, maxCorners=50, qualityLevel=0.01, minDistance=10)
for pt in corners:
   cv2.circle(img, tuple(pt[0]), 5, (0,255,0), -1)
✔️ Practice: Detect corners in a chessboard image, observe how qualityLevel filters weak corners.
💡 Checkboard corner example: Load a chessboard photo → Shi-Tomasi finds inner corners perfectly. Essential for camera calibration!

🔑 Keypoint Detectors: SIFT & ORB

🧬 SIFT (Scale-Invariant Feature Transform)

Patented but now freely available. Robust to scale, rotation, affine changes. Computes 128-dim descriptors.

sift = cv2.SIFT_create()
kp, des = sift.detectAndCompute(gray, None)
# kp = keypoints, des = descriptor array
⚡ ORB (Oriented FAST + Rotated BRIEF)

Efficient, free, and fast. Great for real-time applications. Rotation invariant, binary descriptors.

orb = cv2.ORB_create(nfeatures=1000)
kp, des = orb.detectAndCompute(gray, None)
img_kp = cv2.drawKeypoints(img, kp, None, flags=cv2.DRAW_MATCHES_FLAGS_DRAW_RICH_KEYPOINTS)
🧪 When to use? ORB → real-time, mobile/embedded (panorama stitching, SLAM). SIFT → higher accuracy, varying scales/rotation (photogrammetry, medical).

🔗 Feature Matching: BFMatcher & FLANN

BFMatcher (Brute Force) – compares every descriptor from image1 with all in image2. FLANN – approximate nearest neighbor, faster for large datasets.

💥 BFMatcher + knnMatch
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=False)
matches = bf.knnMatch(des1, des2, k=2)
# Lowe's ratio test to filter good matches
good = [m for m,n in matches if m.distance < 0.75*n.distance]
🏎️ FLANN based Matcher
FLANN_INDEX_KDTREE = 1
index_params = dict(algorithm=FLANN_INDEX_KDTREE, trees=5)
flann = cv2.FlannBasedMatcher(index_params, {})
matches = flann.knnMatch(des1, des2, k=2)
🎯 Practice: Match keypoints between two similar images (e.g., same building from different angles)
Steps: detect ORB → compute descriptors → BFMatcher → drawMatches. Observe inliers/outliers.
# Complete matching example
img1 = cv2.imread('building_left.jpg',0); img2 = cv2.imread('building_right.jpg',0)
orb = cv2.ORB_create()
kp1, des1 = orb.detectAndCompute(img1, None)
kp2, des2 = orb.detectAndCompute(img2, None)
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)
matches = bf.match(des1, des2)
matched_img = cv2.drawMatches(img1, kp1, img2, kp2, matches[:30], None, flags=2)

🌄 Mini Project: Panorama Stitcher (ORB + Homography)

📸 Stitch two overlapping images into a seamless panorama

Pipeline: detect ORB keypoints → match features → compute Homography (perspective transform) → warp image → blend.

import cv2
import numpy as np

def stitch_images(img_left, img_right):
   # 1. Convert to grayscale, detect ORB
   gray1 = cv2.cvtColor(img_left, cv2.COLOR_BGR2GRAY)
   gray2 = cv2.cvtColor(img_right, cv2.COLOR_BGR2GRAY)
   orb = cv2.ORB_create(nfeatures=2000)
   kp1, des1 = orb.detectAndCompute(gray1, None)
   kp2, des2 = orb.detectAndCompute(gray2, None)

   # 2. BFMatcher + Lowe's ratio test
   bf = cv2.BFMatcher(cv2.NORM_HAMMING)
   matches = bf.knnMatch(des1, des2, k=2)
   good_matches = []
   for m, n in matches:
      if m.distance < 0.75 * n.distance:
         good_matches.append(m)

   # 3. Find Homography (need at least 4 matches)
   src_pts = np.float32([kp1[m.queryIdx].pt for m in good_matches]).reshape(-1,1,2)
   dst_pts = np.float32([kp2[m.trainIdx].pt for m in good_matches]).reshape(-1,1,2)
   H, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)

   # 4. Warp left image to right image's plane
   h, w = img_right.shape[:2]
   panorama = cv2.warpPerspective(img_left, H, (w + img_left.shape[1], h))
   panorama[0:h, 0:w] = img_right # simple blend
   return panorama

# usage
left = cv2.imread('left_view.jpg')
right = cv2.imread('right_view.jpg')
result = stitch_images(left, right)
cv2.imwrite('panorama_result.jpg', result)
✅ Expected outcome
Two overlapping photos (30-50% overlap) combined into wide panoramic view.
🧪 Test it: Take photos of your room or landscape. ORB + RANSAC removes mismatched points.
📐 Why Homography? It maps points from one image plane to another, perfect for stitching when camera rotates around its optical center.
⚡ Pro tip: If stitching fails, increase nfeatures or adjust Lowe's ratio threshold (0.7–0.8).

📌 Quick Reference & Best Practices

  • 🔹 Harris/Shi-Tomasi: Fast corner detection — use for tracking, not robust to large scale changes.
  • 🔹 SIFT → patented but extremely robust; ORB → free, fast, good for real-time.
  • 🔹 Matching ratio test (Lowe's): Keep matches where nearest neighbor distance << second nearest.
  • 🔹 RANSAC + Homography: Essential to filter outliers before warping for panoramas.
  • 🔹 Always ensure > 10–20 good matches for stable homography.
