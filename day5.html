📖 Understanding Object Detection: What, Why, How, Where

❓ WHAT is Object Detection?

Identify and localize all objects of interest in an image/video. Output: bounding boxes + class labels (e.g., person, car, dog). Unlike classification, it tells where objects are.

📷 [ person: x=120,y=200,w=80,h=220 ]
🎯 WHY Object Detection?

Autonomous vehicles (pedestrians, cars), surveillance (intruders), medical imaging (tumor detection), retail (shelf analysis), face detection, and interactive systems.

⚙️ HOW does it work?

Two-stage (Faster R-CNN): region proposals → classify each.
One-stage (YOLO, SSD): predict boxes & classes directly in single pass → extremely fast.

🌍 WHERE applied?

Traffic cameras, robotics, retail analytics, sports tracking, augmented reality, wildlife monitoring, industrial quality control.

🧪 Key evolution: Sliding windows (slow, brute-force) → Region proposals (Selective Search) → YOLO (real-time, single neural network).

🔄 Evolution: Sliding Windows → Region Proposals → YOLO

📦 Sliding Windows

Slide a window across image at multiple scales & positions → classify each crop. Extremely slow (~10k+ windows per image), no bounding box regression.

🧩 Region Proposals (R-CNN family)

Generate ~2k candidate regions (Selective Search) → classify each with CNN. Faster R-CNN adds Region Proposal Network (RPN) → end-to-end trainable.

⚡ YOLO (You Only Look Once)

Divide image into grid → each cell predicts bounding boxes + class probabilities. One forward pass → real-time (45-155 FPS). SSD is similar but uses multi-scale feature maps.

# Conceptual difference
# Sliding windows: for each (x,y,scale): crop, classify
# YOLO: CNN outputs SxSx(B*5 + C) tensor → decode boxes in one shot

🚀 YOLO (v5 & v8): Architecture & Pre-trained Models

YOLOv5/YOLOv8 by Ultralytics: user-friendly, high performance, easy to deploy. Pre-trained on COCO dataset (80 classes: person, car, dog, ...).

📦 YOLOv8 (latest)
from ultralytics import YOLO
model = YOLO('yolov8n.pt') # nano, small, medium, large
results = model('image.jpg')
results[0].show() # displays boxes
📦 YOLOv5 (mature)
import torch
model = torch.hub.load('ultralytics/yolov5', 'yolov5s')
results = model('image.jpg')
results.print()
💡 Model sizes: YOLOv8n (nano) → fastest, less accurate. YOLOv8x (xlarge) → slower, highest accuracy. Choose based on your hardware.

👥 Mini Project: Real-Time Person Counter

📸 Detect people in webcam feed and count them in real-time

Goal: Open webcam → run YOLO on each frame → filter only 'person' class → display count on screen → update every frame.

1
Setup & imports – Install ultralytics, opencv-python. Load YOLOv8n pre-trained model.
2
Initialize webcam – cv2.VideoCapture(0) for default camera.
3
Frame loop – read frame → run model detection → filter boxes where class == 0 (person in COCO).
4
Draw bounding boxes & count – draw green rectangles, putText with person count on top-left.
5
Display & exit – cv2.imshow, break on 'q' key press.
# Real-Time Person Counter with YOLOv8
import cv2
from ultralytics import YOLO

# Load model
model = YOLO('yolov8n.pt') # COCO pre-trained

# Open webcam (0 = default camera)
cap = cv2.VideoCapture(0)

while True:
ret, frame = cap.read()
if not ret:
break

# Run YOLO inference
results = model(frame, verbose=False)

# Filter person class (COCO class ID 0 = 'person')
person_count = 0
for box in results[0].boxes:
cls_id = int(box.cls[0])
if cls_id == 0: # person class
person_count += 1
x1, y1, x2, y2 = map(int, box.xyxy[0])
conf = float(box.conf[0])
cv2.rectangle(frame, (x1, y1), (x2, y2), (0, 255, 0), 2)
cv2.putText(frame, f'person {conf:.2f}', (x1, y1-5),
cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0,255,0), 1)

# Display count on frame
cv2.putText(frame, f'People Count: {person_count}', (10, 30),
cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 0, 255), 2)

cv2.imshow('Real-Time Person Counter', frame)
if cv2.waitKey(1) & 0xFF == ord('q'):
break

cap.release()
cv2.destroyAllWindows()
🎥 [WEBCAM FEED] → Detected: 3 people → Bounding boxes drawn → Counter = 3
🧪 Test it yourself: Stand in front of your webcam with friends. The counter updates in real-time. Try with multiple people, partial occlusions — YOLO handles it robustly.
⚡ Optimization tip: For faster FPS, use YOLOv8n (nano) or reduce frame size with cv2.resize().

⚖️ SSD & Faster R-CNN: Alternative Approaches

📐 SSD (Single Shot MultiBox Detector)

Like YOLO but uses multi-scale feature maps for better small object detection. Good speed/accuracy tradeoff. Pre-trained models available in PyTorch.

# torchvision SSD
model = torchvision.models.detection.ssd300_vgg16(pretrained=True)
🐉 Faster R-CNN

Two-stage: Region Proposal Network (RPN) + classifier. Higher accuracy than one-stage, but slower (~5-10 FPS). Best for applications needing precise localization.

model = torchvision.models.detection.fasterrcnn_resnet50_fpn(pretrained=True)
📊 When to choose? YOLO → real-time (30+ FPS), SSD → balanced, Faster R-CNN → max accuracy (medical, satellite). For person counter, YOLOv8 is perfect.

📌 Best Practices for Object Detection

  • 🔹 Confidence threshold: Filter low-confidence detections (e.g., only show boxes with conf > 0.5).
  • 🔹 NMS (Non-Maximum Suppression): Removes duplicate bounding boxes for same object. YOLO applies automatically.
  • 🔹 Input resolution: Higher resolution = slower but better for small objects. YOLO default 640x640 works well.
  • 🔹 GPU acceleration: Use CUDA for real-time: model = YOLO('yolov8n.pt').cuda()
  • 🔹 Class filtering: For person counting, ignore other classes to reduce false positives.
