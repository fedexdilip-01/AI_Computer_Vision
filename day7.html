📖 Model Deployment: What, Why, How, Where

❓ WHAT is Model Deployment?

Taking a trained model and integrating it into a production environment — web app, mobile app, edge device, or cloud API. Making predictions available to end users.

🎯 WHY Optimize & Deploy?

Models are too large/slow for real-time use. Quantization reduces size (4-8x) with minimal accuracy loss. Deployment brings AI to users — real impact!

⚙️ HOW: Optimization + API

Quantization: INT8 vs FP32. Pruning: Remove unnecessary weights. Flask/FastAPI: REST APIs. Streamlit: Interactive dashboards.

🌍 WHERE Deployed?

Web apps (AWS/GCP), mobile (TensorFlow Lite), edge (Raspberry Pi, Jetson), browser (TensorFlow.js), IoT cameras.

🧪 Key insight: A perfect model in Jupyter is useless if it can't run in production. Deployment is the final bridge to real-world value.

⚡ Model Optimization: Quantization & Pruning

📉 Quantization (PyTorch)
import torch
model.eval()
quantized_model = torch.quantization.quantize_dynamic(model, {torch.nn.Linear}, dtype=torch.qint8)
# 75% smaller, minimal accuracy drop
✂️ Pruning (Remove redundant weights)
import torch.nn.utils.prune as prune
prune.l1_unstructured(module, name='weight', amount=0.3)
# Remove 30% of smallest weights → sparsity
📱 TensorFlow Lite
converter = tf.lite.TFLiteConverter.from_saved_model(model_path)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
tflite_model = converter.convert()
📊 Results: ResNet50: 98MB → 25MB (quantized). Speed increase 2-3x on CPU. Perfect for edge deployment.

# Flask API for YOLO object detection
from flask import Flask, request, jsonify
from ultralytics import YOLO

app = Flask(__name__)
model = YOLO('yolov8n.pt')

@app.route('/detect', methods=['POST'])
def detect():
image = request.files['image'].read()
results = model(image)
return jsonify(results[0].boxes.data.tolist())

if __name__ == '__main__':
app.run(host='0.0.0.0', port=5000)

📱 Edge Deployment: TensorFlow Lite & ONNX

🤖 TensorFlow Lite

Run models on mobile (Android/iOS), Raspberry Pi, microcontrollers. Convert → TFLite → delegate GPU/Edge TPU.

interpreter = tf.lite.Interpreter(model_path="model.tflite")
interpreter.allocate_tensors()
interpreter.set_tensor(input_index, input_data)
interpreter.invoke()
🔄 ONNX (Open Neural Network Exchange)

Framework-agnostic. Convert PyTorch/TF → ONNX → deploy anywhere (ONNX Runtime speeds up inference 2-3x).

import onnx
import onnxruntime as ort
session = ort.InferenceSession("model.onnx")
output = session.run(None, {"input": input_numpy})
🌟 Edge devices: Raspberry Pi 4 (YOLOv5s ~5 FPS), Google Coral (Edge TPU ~100 FPS), NVIDIA Jetson Nano.

# app.py - Complete Streamlit Defect Detection App
import streamlit as st
from ultralytics import YOLO
from PIL import Image
import numpy as np

st.set_page_config(page_title="Defect Detection", page_icon="🏭")
st.title("🏭 Manufacturing Defect Detection")
st.write("Upload a product image to detect defects (scratches, dents, missing parts)")

@st.cache_resource
def load_model():
return YOLO('model/best.pt')

model = load_model()
uploaded_file = st.file_uploader("Choose an image...", type=['jpg', 'png', 'jpeg'])

if uploaded_file is not None:
image = Image.open(uploaded_file)
st.image(image, caption='Uploaded Image', use_container_width=True)

if st.button('Detect Defects'):
with st.spinner('Analyzing...'):
results = model(np.array(image))
annotated = results[0].plot()
st.image(annotated, caption='Detection Result', use_container_width=True)

defect_count = len(results[0].boxes)
if defect_count > 0:
st.error(f"⚠️ {defect_count} defects detected!")
else:
st.success("✅ No defects detected. Product OK.")

st.markdown("---")
st.markdown("🚀 Deployed with YOLOv8 + Streamlit")

📌 Deployment Best Practices

  • 🔹 Always quantize before deploying to CPU/edge devices → 4x smaller, 2x faster.
  • 🔹 Use caching (st.cache_resource in Streamlit) to load model once.
  • 🔹 Set confidence thresholds appropriately (0.5 for general detection, 0.7 for high precision).
  • 🔹 Environment variables for API keys when using cloud services.
  • 🔹 Test with Docker locally before cloud deployment to avoid "works on my machine" issues.
  • 🔹 Monitor model drift in production — retrain periodically with new data.
